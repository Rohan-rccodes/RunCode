What is PRIMARY KEY constraint And how it can be use in column level contraints (CLC)?

~The PRIMARY KEY constraint uniquely identifies each record in a table. Primary keys must contain UNIQUE values, and cannot contain NULL values.
~A table can have only ONE primary key; and in the table, this primary key can consist of single or multiple columns (fields).
~The PRIMARY KEY constraint is simply a combination of NOT NULL and UNIQUE constraints.
Meaning, the column cannot contain duplicate as well as NULL values.

Syntax:-

create table table_name (col1 datatype primary key,col2 datatype,.................);
