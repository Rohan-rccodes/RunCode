SUB-QUERY

      Nested query is a query within another query. SQL has an ability to nest queries within one another. A subquery is a SELECT statement that is nested within another SELECT statement and which returns intermediate results. The result of subquery or inner query is dynamically substituted in the condition of outer query. SQL first evaluates the inner query or subquery within the WHERE clause. The result of inner query is then substituted in the condition of the outer query

Most of the time, a subquery is used when you know how to search for a value using a SELECT statement, but do not know the exact value.

Advantages of Nested queries:-

• Sub queries allow a developer to build powerful commands out of simple ones.
• The nested subquery is very useful when you need to select rows from a table with a condition that depends on the data in the table itself.

Example:-

1. Find out the name of the student who is having highest marks?

2. To out names of the student who is second in the class.
